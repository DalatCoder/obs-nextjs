## Thiết Lập GitHub Repository cho Deployment

### Tổng quan

Bước đầu tiên để thiết lập **Professional Continuous Deployment** (triển khai liên tục chuyên nghiệp) với Vercel là tạo GitHub Repository chứa các file dự án của bạn.

### Git Repository tự động từ Create-Next-App

Khi khởi tạo dự án với Create-Next-App, công cụ này đã tự động tạo một **Git Repository** (kho mã nguồn Git) cho dự án. Bạn có thể nhận biết điều này qua các dấu hiệu sau:

- Các file có marker (dấu hiệu) là "modified" (đã sửa đổi) hoặc "untracked" (chưa được theo dõi)
- Thanh trạng thái phía dưới hiển thị đang ở nhánh "main branch"


### Các bước thực hiện

#### 1. Thêm và commit các file

**Thêm tất cả file vào staging area:**

```bash
git add .
```

**Commit các file:**

```bash
git commit -m "Initial Commit"
```

Sau khi commit, staging area sẽ trống và không còn marker nào hiển thị trên các file.

#### 2. Tạo GitHub Repository mới

Truy cập GitHub và tạo repository mới với các thiết lập:

- **Tên repository**: `the-wild-oasis-website` (hoặc tên phù hợp với dự án của bạn)
- **Visibility**: Private (riêng tư)
- **Không chọn**: README, .gitignore hoặc license (vì dự án đã có sẵn các file này, việc thêm sẽ gây xung đột)


#### 3. Kết nối repository local với GitHub

Sau khi tạo repository trên GitHub, sao chép các lệnh được cung cấp:

```bash
git remote add origin <URL-repository>
git push -u origin main
```

Các lệnh này sẽ:

- Thêm remote branch (nhánh từ xa)
- Push (đẩy) code từ repository local lên GitHub

**Lưu ý**: Quá trình push có thể mất thời gian nếu dự án có nhiều hình ảnh hoặc file lớn.

#### 4. Xác nhận

Sau khi hoàn tất, reload trang GitHub và bạn sẽ thấy toàn bộ code đã được đẩy lên thành công.

### Bước tiếp theo

Với repository đã được thiết lập trên GitHub, bạn có thể chuyển sang bước kết nối repository này với Vercel để triển khai ứng dụng.
