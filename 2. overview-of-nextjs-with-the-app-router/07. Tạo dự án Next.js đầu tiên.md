## Tạo dự án Next.js đầu tiên

### Khởi tạo dự án

Để tạo một dự án Next.js mới, sử dụng lệnh tương tự như khi tạo dự án React với Vite hoặc Create React App. Mở terminal trong thư mục mong muốn và chạy lệnh:

```bash
npx create-next-app@14 the-wild-oasis-website
```

Lưu ý quan trọng: Khóa học sử dụng Next.js version 14 để đảm bảo tính nhất quán. Next.js phát hành phiên bản mới mỗi năm, nên việc sử dụng cùng version giúp mọi thứ hoạt động giống hệt như trong video.

### Cấu hình dự án

Trong quá trình khởi tạo, create-next-app sẽ hỏi một số câu hỏi cấu hình. Sử dụng các tùy chọn sau:

- **TypeScript:** No (để mọi người có thể hiểu dự án mà không cần biết TypeScript)
- **ESLint:** Yes
- **Tailwind CSS:** Yes (Tailwind được tích hợp chặt chẽ vào Next.js)
- **src/ directory:** No
- **App Router:** Yes (cách được khuyến nghị hiện nay)
- **Import alias:** No


### Cấu trúc thư mục dự án

Sau khi cài đặt hoàn tất, dự án sẽ có cấu trúc sau:

- **app/**: Thư mục chính cho App Router. Sự hiện diện của thư mục này kích hoạt App Router
- **public/**: Chứa static assets (tài nguyên tĩnh) để phục vụ người dùng
- **package.json**: Vì Next.js về bản chất là một Node.js app (cần server), file này quản lý dependencies
- **next.config.js**: File cấu hình Next.js (ban đầu trống)
- **tailwind.config.js**: Cấu hình Tailwind CSS
- **.eslintrc.json**: Quy tắc ESLint

Next.js cũng tự động tạo Git repository mới cho dự án.

### Tạo page đầu tiên

Trong thư mục `app/`, xóa các file mặc định (CSS files, icon) nhưng giữ lại `page.js`. Đây là file quan trọng nhất. Cập nhật nội dung như sau:

```javascript
export default function Page() {
  return <h1>Hello Next!</h1>;
}
```

Đây chỉ là một React component thông thường được export như default function từ `page.js`.

### Chạy development server

Để khởi động ứng dụng, sử dụng lệnh:

```bash
npm run dev
```

Lệnh này sẽ chạy development server. Mở trình duyệt tại địa chỉ hiển thị trong terminal (thường là `http://localhost:3000`).

### Kiểm tra Server-Side Rendering

Để xác minh ứng dụng đang sử dụng server-side rendering:

1. Mở trang web trong trình duyệt
2. Nhấn chuột phải chọn "View Page Source" (Xem nguồn trang)
3. Tìm thẻ `<h1>Hello Next!</h1>` trong HTML source

Nếu thấy nội dung này trong page source (không phải trong DevTools Elements), điều đó chứng tỏ HTML đã được generate (tạo ra) trên server và gửi đến browser. Khác với React app thông thường, nơi nội dung chỉ xuất hiện trong DOM sau khi JavaScript chạy.

### Ghi chú thêm

Dự án **The Wild Oasis Website** sẽ được xây dựng xuyên suốt phần 5 của khóa học. Đây là customer-facing website (trang web hướng đến khách hàng) dựa trên dữ liệu từ dự án Wild Oasis trước đó, cho phép khách hàng:

- Xem tất cả các cabin (nhà nghỉ)
- Tạo reservations (đặt phòng)
- Quản lý account (tài khoản)
- Thay đổi profile data (dữ liệu hồ sơ)
- Xóa và cập nhật reservations của họ

Nếu muốn sử dụng Pages Router thay vì App Router, cần tạo thư mục `pages/` ở root level thay vì `app/`, nhưng điều này không được khuyến nghị cho dự án mới.
